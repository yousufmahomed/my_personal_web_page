Sales Coordination
CRM Architecture

Implementing a Least Privilege Security Model & Intelligent Automation

Project Overview

This project demonstrates the architectural design of a custom Salesforce CRM for high-end auto shop sales. The objective was to design a secure environment utilizing a "Least Privilege" data model while automating the hand-off process between Appointment Setters and Sales Closers.

Salesforce Flow Builder Security & Visibility Data Governance Permission Sets Validation Rules

Automation Intelligence

To bridge the gap between security and efficiency, I engineered a Record-Triggered Flow that manages the lead hand-off. When a setter marks a lead for review, the system automatically identifies the owner and fires a real-time push notification.

Architecting the Logic

Process: The Flow triggers when a Lead Status changes to 'Pending Closer Review'. It retrieves the Custom Notification metadata, assigns the Recipient ID (Record Owner), and executes the push action with a deep-link to the record.

The Front-End: Custom Lead Path with the 'Pending Closer Review' stage successfully integrated.

Security & Governance

1. Role-Based Permission Sets

Financial fields like Final Quotation Price and Discount Amount are hidden at the profile level. Access is granted exclusively to "Sales Closers" via custom Permission Sets, ensuring sensitive margin data remains protected.

Adjustment of picklist to cater to cold and hot leads

2. Process Guardrails

Application of discounts as authorized by sales manager and not appointment setters

To enforce data integrity, a Validation Rule prevents Appointment Setters from bypassing the hand-off process. This ensures all high-value builds are reviewed by a manager before being finalized.

Development of custom notifications for use in flow path.
